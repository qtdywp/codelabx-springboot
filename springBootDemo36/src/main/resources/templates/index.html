Endpoints清单

方法 路径 描述 查看路径
GET /auditevents 显示应用暴露的审计事件（如认证进入、订单失败） /actuator/auditevents
GET /beans 描述应用程序上下文里全部的 Bean 以及它们的关系 /actuator/beans
GET /conditions 就是1.0的/autoconfig，提供一份自动配置生效的条件情况，记录哪些自动配置条件通过了，哪些没通过 /actuator/conditions
GET /configprops 描述配置属性（包含默认值）如何注入 Bean /actuator/configprops
GET /env 获取全部环境属性 /actuator/env
GET /env/{name} 根据名称获取特定的环境属性值 /actuator/env/java.vm.version
GET /flyway 提供一份 Flyway 数据库迁移信息 /actuator/flyway
GET /health 报告应用程序的健康指标，这些值由HealthIndicator的实现类提供 /actuator/health
GET /httptrace 显示 HTTP 足迹，最近 100 个 HTTP request/repsponse /actuator/httptrace
GET /info 获取应用程序的定制信息，这些信息由 info 打头的属性提供 /actuator/info
GET /loggers 显示和修改配置的 loggers /actuator/loggers
GET /liquidbase 显示 Liquibase 数据库迁移的纤细信息 /actuator/liquidbase
GET /metrics 报告各种应用程序度量信息，比如内存用量和 HTTP 请求计数 /actuator/metrics
GET /metrics/{name} 报告指定名称的应用程序度量值 /actuator/metrics/mem.free
GET /mappings 描述全部的 URI 路径，以及它们和控制器（包含 Actuator 端点）的映射关系 /actuator/mappings
GET /scheduledtasks 展示应用中的定时任务信息 /actuator/scheduledtasks
GET /sessions 如果我们使用了 Spring Session 展示应用中的 HTTP Sessions 信息 /actuator/sessions
POST /shutdown 关闭应用程序，要求 endpoints.shutdown.enabled 设置为 true curl -X POST "http://localhost:8080/actuator/shutdown"
GET /threaddump 获取线程活动的快照 /actuator/threaddump

如果您的应用程序是web应用程序（Spring MVC、Spring WebFlux或Jersey），则可以使用以下Endpoints：

方法 路径 描述 查看路径
GET /heapdump dump一份应用的JVM堆信息 /actuator/heapdump
GET /jolokia 展示JMX beans（仅当Jolokia配置到classpath的情况下可用。不可用于WebFlux） /actuator/jolokia
GET /logfile 返回 log file 中的内容（如果 logging.file 或者 logging.path 被设置 /actuator/logfile
GET /prometheus Exposes metrics in a format that can be scraped by a Prometheus server. /actuator/prometheus
